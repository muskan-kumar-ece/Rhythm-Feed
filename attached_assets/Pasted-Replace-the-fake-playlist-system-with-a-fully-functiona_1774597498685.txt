Replace the fake playlist system with a fully functional real playlist feature.

---

🎯 PROBLEM:

- Playlist tab shows hardcoded data
- Users cannot create, edit, or manage playlists
- No real backend connection

---

🛠️ STEP 1: DATABASE SCHEMA

Create tables:

1. playlists
- id (uuid)
- userId (owner)
- name
- description
- coverImage (optional)
- createdAt

2. playlistSongs
- id
- playlistId
- songId
- addedAt

---

🛠️ STEP 2: BACKEND ROUTES

Create APIs:

1. POST /api/playlists
→ create playlist

2. GET /api/playlists
→ get user playlists

3. GET /api/playlists/:id
→ get playlist with songs

4. POST /api/playlists/:id/add-song
→ add song to playlist

5. DELETE /api/playlists/:id/remove-song/:songId
→ remove song

6. DELETE /api/playlists/:id
→ delete playlist

7. PATCH /api/playlists/:id
→ update name/description

---
